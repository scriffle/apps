About the evidence

      
    
        
  • Effect sizes are not promises. They depend on the comparison, the measure, and the delay before testing.
    • 
        
  • Most evidence is lab-based, short-interval and near-transfer. Classroom and delayed effects are usually smaller, so do not overstate durability.
    • 
        
  • Mechanism over brand. The learning comes from retrieval, spacing and elaboration, not the game wrapper.
    • 
        
  • Implementation dominates. A poorly run version of any activity can produce no gain.
    • 
      

      
Activities we avoid, and what to use instead
        
    
          
  • Learning styles (VAK/VARK). No support. Use dual coding done well, plus retrieval.
    • 
          
  • Pure unguided discovery for novices. Less effective. Use worked then faded examples with self-explanation.
    • 
          
  • Gamification for its own sake. Engagement without reliable learning. Use a game where correct content is the path to progress, plus retrieval.
    • 
          
  • Jigsaw as a guaranteed booster. Inflated effects. Use a structured cooperative model with individual accountability, or peer instruction.
    • 
          
  • Brand quiz tools as proven (Blooket, Gimkit, Quizizz). Thin evidence. Build the questions as a retrieval routine, or label them engagement-oriented.
    •

How to share with students

    

      
Claude builds you a single HTML resource, or a print or CSV file. Here is how to get it to your students, easiest first. You do not need your own website.

      
    
        
  1. Use a platform you already have. For quizzes and flashcards, the export formats here produce a Form, a table or a file to import into Google Forms, Quizlet, Blooket, Wayground or Anki, which then host the activity for your students, with a normal link and no student account. Print resources just print. The most school-safe routes.
    2. 
        
  2. Publish it as a Claude artifact. In Claude, open the resource it built, click Publish, and copy the link. Students open it in a browser with no account and it runs. Catch: a published artifact cannot save progress, so spaced-repetition resets, and a school-managed Team or Enterprise Claude account cannot publish publicly, use a personal Free, Pro or Max account.
    3. 
        
  3. Drop the file on Tiiny.host. Go to tiiny.host, drag the .html in, and get a link. This is the option that keeps saved progress, because it is a real web address. Catch: make a free account and log in about monthly to keep the link alive.
    4. 
        
  4. Google Sites (Workspace schools). New Site, then Insert, Embed, Embed code, paste the whole file, Publish, and share the link. It runs on sites.google.com, which school filters allow. Catch: very large files may be rejected by the embed box.
    5. 
        
  5. Send the file itself through Google Classroom or email as a last resort when sites are blocked. Catch: it will not run on iPhone or iPad, and there is no link, give phone users a link instead.
    6. 
      

      
Saved progress (spaced repetition) only survives from a real web link such as Tiiny.host or Google Sites, not from a published artifact and not reliably from a downloaded file, and it never syncs between devices. Google Drive cannot run the tool, it only shows the code or downloads it. Test your link on a student device before the lesson, school networks block some sites.
